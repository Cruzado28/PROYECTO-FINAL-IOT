Reglas del acceso

El lector de acceso registra entrada o salida. Si el vehículo ya está dentro, no puede volver a ingresar. Si intenta salir sin pago, se registra una incidencia.

Pago separado

El segundo RFID solo procesa el pago y actualiza el saldo de la tarjeta. Luego el lector de acceso permite la salida.

RFID de acceso y RFID de pago — RC522 / NTAG215

Este lector decide si el vehículo ingresa o sale. Si intenta ingresar dos veces o salir sin pago, se registra una incidencia.

RFID de pagoCobro y actualización de saldo · RC522
SS: GPIO 17RST: GPIO 16SCK: GPIO 18MISO: GPIO 19MOSI: GPIO 23
Sin conexión al ESP32

Este lector calcula el pago por los minutos reales transcurridos, descuenta saldo y deja la tarjeta habilitada para salir.

Esperando lectura…Acerca una tarjeta al lector de acceso o pago
Datos para la tarjetaSe escribirán desde el RFID de pago

Procedimiento

  1. Completa los datos del vehículo y saldo inicial.
  2. Presiona “Escribir tarjeta”.
  3. Acerca la NTAG215 al lector RFID de pago.
  4. En funcionamiento, el lector de acceso registra entrada/salida y el lector de pago descuenta el saldo.

Importante: si el vehículo intenta salir por el lector de acceso sin haber pagado, se genera una incidencia para reportes.
